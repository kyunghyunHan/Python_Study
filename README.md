## 인터프리터와 컴파일러의 차이점

- 인터프리터와 컴파일러는 모두 프로그래밍 언어를 실행 가능한 기계어로 변환하는 역할을 한다.

- 하지만 이 두 가지 방식에는 몇 가지 차이점과 공통점이 있다.

- 먼저 공통점으로는 프로그램 코드를 실행 가능한 기계어로 변환하는 것이다.
- 이를 통해 프로그래밍 언어를 사용하여 작성된 코드를 실행할 수 있게 된다.
- 또한, 인터프리터와 컴파일러는 모두 프로그램 코드를 실행하기 전에 일종의 검사과정을 거쳐 문법 오류 등을 확인한다.

- 하지만 두 방식의 가장 큰 차이점은 프로그램 코드를 변환하는 시점이다.
- 컴파일러는 프로그램 코드를 실행하기 전에 먼저 전체 코드를 분석하고, 이를 바탕으로 실행 가능한 기계어로 번역한다.
- 이 과정에서 일반적으로 실행 파일이 생성되며, 이 파일을 실행하면 기계어로 번역된 코드가 실행된다.
- 이 방식은 실행 속도가 빠르다는 장점이 있다.

- 반면 인터프리터는 프로그램 코드를 한 줄씩 읽어들이면서 즉시 실행 가능한 기계어로 번역한다.
- 이 과정에서 실행 파일이 생성되지 않는다.
- 인터프리터는 실행 파일을 생성하지 않기 때문에, 프로그램 코드를 수정할 때마다 다시 번역하는 작업이 필요하지 않는다.
- 이 방식은 실행 속도가 느리다는 단점이 있다.

- 또한, 컴파일러는 대부분의 경우 언어의 특정 버전에 대해 지원이 되는 반면, 인터프리터는 일반적으로 다양한 버전의 언어를 지원한다.
- 이는 인터프리터가 코드를 실행할 때마다 번역을 수행하기 때문이다.

- 마지막으로, 인터프리터는 대화형 모드(interactive mode)를 지원하는 경우가 많다.
- 이는 사용자가 코드를 입력하면 즉시 실행 가능한 기계어로 번역하여 결과를 표시하는 모드이다.
- 이 방식은 테스트나 디버깅 작업 등에 유용하다.

- 따라서, 인터프리터와 컴파일러는 모두 프로그래밍 언어를 실행 가능한 기계어로 변환하는 공통점이 있지만, 프로그램 코드를 변환하는 시점이 다르며, 이에 따라 실행 속도나 언어 버전 지원 등에 차이가 있다.
